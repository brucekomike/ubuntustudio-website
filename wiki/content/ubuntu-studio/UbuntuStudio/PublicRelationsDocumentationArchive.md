The purpose of this wiki page is to define the public relations strategy for the Ubuntu Studio Public Relations team. This will include defining the team, the [communications channels](PublicRelationsDocumentation#Definitions) used, list of events to publicize, etc.

## PR Strategy

The primary goals of Ubuntu Studio Public Relations is to:

- communicate more often to users
- encourage developers to join the Ubuntu Studio Development Team
- calls for volunteers for testing

And the following means will be used to accomplish these goals:

- establish a PR team
- assign some specific responsibilites to specific members
- create an "event vs channel" matrix to show when and where to post
- provide "standard announcement verbiage" templates for various events
- use ubuntustudio.org as the nexus or focal point of our PR communications

## Public Relations Team

The Ubuntu Studio Public Relations Team will be primarily repsonsible for all public relations and external (i.e. not inter-team or to Ubuntu) communications for Ubuntu Studio.

To become a member of the PR team, first one must volunteer either in [#ubuntustudio-devel on Freenode IRC](http://webchat.freenode.net/?channels=ubuntustudio-devel) or on the [ubuntustudio-devel mailing list](https://wiki.ubuntu.com/ubuntu-studio-devel%40lists.ubuntu.com) and then be approved by one of the PR team's admins.

The team should have at least two admins to ensure responsiveness to volunteers and other matters for the team.

The [Ubuntu Studio Public Relations Team](https://launchpad.net/~ubuntustudio-public-relations) will act as the Launchpad team for the PR team, providing a mechanism for controlling access to the website (i.e. authentication for website login has been assigned to the website team).

The PR team should remain small, perhaps no more than five active members. This should hopefully prevent major overlap of news coverage and hopefully such a small group can find an easy (and informal) consensus about what topic each member should cover.

Furthermore, it might be wise to assign one PR team member for some large, specific events (e.g. release) with a secondary as fail over.

### How to Post

Use only text, when writing posts. Save the article as a textfile. Then post the same text to each channel, one at a time. Use the #ubuntustudio hashtag in footer. (!ubuntustudio on identi.ca).

- **Website & Social Sites:** Post an article on the [Ubuntu Studio Website](http://ubuntustudio.org/wp-admin). This post is published on facebook, twitter, identi.ca and the website at once.
- **G+:** Have Post published on the [Ubuntu Studio G+ Page](https://plus.google.com/b/102125777892703446963/)
- **mail lists:** Post to all mail lists at once: [ubuntu-studio-users@lists.ubuntu.com](mailto:ubuntu-studio-users@lists.ubuntu.com), [linux-audio-user@lists.linuxaudio.org](mailto:linux-audio-user@lists.linuxaudio.org), [community-announce@lists.ubuntu.com](mailto:community-announce@lists.ubuntu.com)
- **forums:** Visit the [Ubuntu Studio Section](http://ubuntuforums.org/forumdisplay.php?f=335) at ubuntuforums.org and create a new post. Visit the [Linux Music News](http://www.linuxmusicians.com/viewforum.php?f=24&sid=70c46b62dce56baa85c57b8f14d82134) section at linuxmusicians.com and create a new post.
